# AutoLabel v1.0.0

AutoLabel v1.0.0 is the first public research release of the project.

## What it is

AutoLabel is an autonomous weak-supervision system that uses an LLM to generate, validate, and iteratively improve labeling functions. The system keeps only LF batches that improve held-out dev F1 and produces inspectable Python rules rather than opaque prompt-only behavior.

## What is included

- autonomous LF generation with a keep/discard ratchet
- multiple label models
- Anthropic, OpenAI, Groq, and Ollama providers
- multilingual prompting support with Hindi and Marathi loaders
- measured benchmark tooling
- publication-style visualization CLI
- free-tier guardrails for LLM benchmark baselines

## Featured measured result

On the bundled `airline_tweets` entity-extraction benchmark:

- `TF-IDF + LogReg`: `0.784` test F1
- `AutoLabel`: `0.656` test F1

This release does not claim supervised-baseline wins. The release claim is that AutoLabel automates LF authoring and produces a measurable autonomous weak-supervision workflow with committed proof artifacts.

## Limitations

- benchmark methodology still uses labeled train and dev splits
- multilingual support is promising but not yet benchmarked as a broad superiority claim
- full end-to-end benchmark sweeps with every LLM baseline may be slow on free tiers
